# 上海公共设施采集计划

## 目标

生成一次性上海公共设施快照，覆盖博物馆、美术馆、市区及街镇图书馆、上海行政区内运营地铁站、三甲医院、二甲医院、社区卫生服务中心、分中心与服务站。

## 核心原则

- 官方名录决定名称、类别和医院等级；高德Web服务API补充POI ID、地址和GCJ-02坐标。
- 高德MCP只用于人工处理重名、迁址和多候选记录，自动流程不依赖MCP。
- 每个实际地点一行；不同院区、分馆、分中心和服务站不得自动合并。
- 三乙、二乙、诊所、护理院、在建站和上海市外站点不在范围内。
- 高德坐标必须始终标记为`GCJ-02`，不得标为WGS84。
- 高德搜索默认每秒最多一次；遇到`CUQPS_HAS_EXCEEDED_THE_LIMIT`时按2、4、6秒退避重试，并依赖每10条写入的检查点继续运行。

## 数据流程

1. 下载或导入官方网页、PDF、Excel快照，保留来源URL、日期和文件哈希。
2. 将各来源规范化为统一的原始来源记录。
3. 用官方名称和地址调用高德POI搜索；找不到时调用地理编码。
4. 按名称、地址、行政区和POI类型评分匹配，低置信度进入复核队列。
5. 去重、空间校验、来源完整性校验后，导出Excel、CSV和GeoJSON。

## 数据源优先级

1. 上海市文旅局、卫健委和市政府正式文件。
2. 区政府、区卫健委、医保局公开目录。
3. 地铁运营方公开线路资料。
4. 高德POI API。
5. 高德MCP和天地图人工核验。

## 验收

- 医院等级必须有官方来源。
- 每条最终记录必须至少有一个来源记录。
- 自动匹配必须保留高德POI ID和匹配分数。
- 换乘站只能有一条设施记录，并通过单独关系表关联线路。
- 最终输出中不能出现`rejected`记录；`map_only`必须在质量报告中单独统计。
